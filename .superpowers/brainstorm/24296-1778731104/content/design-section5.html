Brio Theme — Implementation Plan

Section 5/5: Phases and files to create/modify

Phase 1: Database + Backend API

  • Migration: 0020_brio_theme.sql — create store_theme_settings table
  • Schema: Add storeThemeSettings to apps/backend/src/db/schema.ts
  • Route: GET /api/v1/public/stores/:slug/theme in public scope
  • Route: PUT /api/v1/merchant/theme in merchant scope
  • Service: theme.service.ts — CRUD + Redis cache

  • Routes: +layout.svelte, +page.svelte (home), menu/+page.svelte, menu/[slug]/+page.svelte, product/[slug]/+page.svelte, cart/+page.svelte, checkout/+page.svelte, contact/+page.svelte
  • Shared layer: apps/storefront-food/src/lib/shared/ — move existing API/types/cart there

Phase 4: Routing + Integration

  • Theme detection: store's store_theme_settings.theme_name
  • Route: /store/{slug} checks theme and redirects to /store/{slug}/brio
  • Classic theme moved to apps/storefront-food/src/lib/themes/classic/
  • Build passes, TypeScript 0 errors

Phase 5: Verification

  • pnpm typecheck — 0 errors

  • Test: Merchant switches theme in dashboard → storefront updates
  • Test: Add to cart, checkout flow works in Brio theme
