## Prioritization

1. **P0 Critical** — Security risks, data loss, production crashes, race conditions
2. **P1 High** — Bugs affecting user experience, performance degradation, authorization gaps
3. **P2 Medium** — Code quality, missing tests, minor optimizations, dead code

## Test Strategy

For each fix:
1. Write a failing test that reproduces the bug/vulnerability
2. Implement the fix
3. Verify the test passes
4. Run existing test suite to check for regressions
5. For security fixes: add specific security test cases

**Critical test coverage needed:**
- Concurrent request tests for race conditions (use `Promise.all` with 2+ parallel requests)
- Cross-tenant access tests (request data from another store's resources)
- XSS payload tests for storefront JSON-LD
- Cart ownership verification tests
- Payment duplicate prevention tests
